Image Arithmetic Subtraction

Explore how images are combined using pixel-by-pixel subtraction. Subtracting pixel values can reveal differences between images, enhance edges, and support background removal.

How Image Subtraction Works

Image arithmetic subtraction combines two images by subtracting corresponding pixel values. Each pixel in the result is computed as the value of Image A minus Image B at the same coordinate.

The Formula

Result(x,y) = ImageA(x,y) − ImageB(x,y)

• x,y are the pixel coordinates

• Results are clamped to the 0–255 range

• Applied independently to R, G, and B channels

Use Cases

Change Detection: Highlight differences between two frames.

Background Removal: Subtract a static background from a scene.

Edge Emphasis: Enhance edges by attenuating uniform regions.

Mathematical Insights

Properties of Subtraction

Non‑Commutative: A − B ≠ B − A (order matters)

Non‑Associative: (A − B) − C ≠ A − (B − C)

Identity: A − 0 = A (0 is the black image)

Inverse: A − A = 0 (black image)

Relation to Difference: |A − B| emphasizes magnitude without sign; useful for change maps.

Common Pitfalls

Underflow: Negative results are clamped to 0, losing sign information.

Order Sensitivity: Swapping A and B flips the sign; choose consistently.

Noise Amplification: Subtraction can amplify sensor noise and misalignments.

Gamma/Color Space: Operate in linear space; sRGB subtraction can be misleading.

Registration: Misaligned images produce false edges; align before subtracting.
